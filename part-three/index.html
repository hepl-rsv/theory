Caractéristiques techniques

d'un serveur

L'efficacité d'un serveur s'évalue sur trois points

Son débit

Sa disponibilité

Sa scalabilité

Serveurs › caractéristiques techniques

Débit

Capacité à recevoir/traiter/envoyer
une certaine quantité de données par unité de temps.

La quasi-totalité des composants de la machine influent.

C'est pourquoi les serveurs sont souvent des machines
dotées de composants puissants.

Serveurs › caractéristiques techniques

Disponibilité

Capacité à fonctionner sans interruption
ou presque.

Plusieurs mécanismes sont mis en place pour accroître cette disponibilité :
  • Redondance matérielle
  • Composant remplaçable à chaud
  • CDN
  • …
Serveurs › caractéristiques techniques › disponibilité

On parle d'

Uptime

pour désigner le laps de temps depuis lequel un système
est allumé sans interruption

Il existe des concours d'uptime informels…

Il n'est pas rare de voir des uptimes de plusieurs
centaines de jours sur des serveurs

Record absolu : 16 ans pour un serveur sous Novel NetWare
Serveurs › caractéristiques techniques

Scalabilité

Anglicisme intraduisible

Capacité à s'adapter à une augmentation de la charge

Serveurs › caractéristiques techniques › scalabilité

On parle de

Scalabilité verticale

quand on augmente les possibilités de la machine

Serveurs › caractéristiques techniques › scalabilité

On parle de

Scalabilité horizontale

quand on augmente le nombre de machines
pour répartir la charge

Serveurs › caractéristiques techniques › scalabilité

On parle enfin de

Scalabilité architecturale

quand le système a été conçu dès le départ
pour se répartir facilement sur plusieurs machine

sans trop générer de frais supplémentaires

Composants matériels

d'un serveur

Tout ordinateur peut être un serveur :
  • un insignifiant Raspberry π (Mono-700MHz/256Mo RAM)
  • un honorable Xeon E5 (8 core à 3,7GHz/128Go RAM)
  • au godzillesque Bullion S (288 cœurs Xeon E7/4To RAM)
Serveurs › composants

Deux écoles de pensées

Investir dans du matériel robuste et puissant qui tiendra sur le long terme
mais ne pourra pas forcément évoluer énormément

Investir dans une architecture plus modeste et moins fiable
mais plus facilement remplaçable/adaptable

Serveurs › composants

Processeurs

en simplifiant très fort

Un cœur/thread ne peut traiter qu'un information à la fois.
Multiplier les cœurs/processeurs permet de gérer
plus d'informations simultanément...

Pour peu que le système puisse le gérer
Serveurs › composants

La RAM

zone mémoire extrêmement rapide dans la laquelle la machine
place les données en cours de traitement

un serveur a tendance à consommer beaucoup de RAM
Serveurs › composants › RAM

Un mécanisme appelé mémoire SWAP permet d'écrire sur le disque
l'excédent de RAM en attente de traitement. Lire/écrire dans la mémoire SWAP
est une opération coûteuse en performance.

mais attention…
Serveurs › composants › RAM

Avoir beaucoup de RAM disponible…

…ne doit pas être une excuse à une mauvaise optimisation !

Serveurs › composants

Disques durs

en fonction des services fournis par le serveur,
l'espace disque et la vitesse d'écriture est un aspect à considérer

La redondance des données peut être assurée par la mise en place
d'un système RAID (hardware ou software)

Serveurs › composants

Bande passante

on peut avoir une machine de brutasse,
si la bande passante ne suit pas, on se retrouve avec un
Goulot d'étranglement
qui limite les performances globales du système

Serveurs › composants

Alimentation

Point critique, l'alimentation doit être stable, rentable,
capable d'encaisser les variations de tension (parafoudre),
et les éventuelles coupures d'énergies (onduleurs)

Serveurs › composants

Refroidissement

Un serveur produit pas mal de chaleur, qu'il faut pouvoir évacuer.

Ces dernières années, de nombreuses iniatives intéressantes ont pris forme :
  • Recyclage de la chaleur produite par les data-center pour chauffer des batiments
  • Data-center en pression négative
  • …

Choisir un serveur › hébergement

Auto-hébergement

une machine domestique, accessible depuis l'extérieur

  • Avantages :
    peu cher, permet de recycler du matos, pratique pour le développement et le testing
  • Inconvénients :
    bande passante limitée par la connexion (et partagée par les autres membres du réseau), consommation électrique, disponibilité difficilement garantie, configuration à la main
Choisir un serveur › hébergement

Hébergement mutualisé

portion d'un serveur physique (hdd, trafic, bdd)
qui est allouée au client, qui partage la machine
avec d'autres clients

  • Avantages :
    peu cher, configuration aisée, support relativement accessible
  • Inconvénients :
    parfois peu évolutif, configurations limitées, services exotiques souvent verrouillés
Choisir un serveur › hébergement

VPS (Virtual Private Server)

(la plupart du temps),
petite machine virtuelle possédant un système dédiée,
partageant certaines ressources tout en se gérant seul

  • Avantages :
    relativement abordable, indépendance des ressources, liberté de configuration
  • Inconvénients :
    configuration matérielle souvent fermée, configuration logicielle à la main, bande passante parfois limite
Choisir un serveur › hébergement

Serveur dédié

machine physique parquée dans un data-center,
dont l'administration interne est entièrement à votre charge

  • Avantages :
    indépendance complète, liberté de configuration matérielle et logicielle
  • Inconvénients :
    budget conséquent, support technique logiciel limité ou payant, configuration à la main
Choisir un serveur

Le choix de l'hébergement doit se faire en fonction de trois critères

  • les spécifications techniques
  • le budget
  • les ressources humaines
Choisir un serveur › critères de choix

vérifier les specs de l'hébergeur
et s'assurer que votre site tournera dessus

il peut être utile de dupliquer la config localement avec une VM
Choisir un serveur › critères de choix

budget

le coût de l'hébergement doit être anticipé
pour éviter les mauvaises blagues

pour les petits clients, ce sera souvent le critère déterminant
